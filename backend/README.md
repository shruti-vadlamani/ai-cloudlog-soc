# Cloud SOC Backend API

FastAPI backend for the AI-Driven Cloud SOC dashboard with **integrated pipeline orchestration**.

## Features

- **Alert Management**: Query and filter ensemble alerts
- **RAG-Powered Enrichment**: Get MITRE techniques, playbooks, and context for each alert
- **Knowledge Base Queries**: Search threat intelligence and behavioral incidents
- **Statistics Dashboard**: Overview metrics, model performance, timelines
- **Playbook Library**: Access incident response playbooks with investigation steps
- **🚀 Pipeline Orchestration**: Automated CloudTrail ingestion, ML model execution, and alert generation

## Production Pipeline

The backend now includes a **fully automated pipeline** that:

1. **Reads real CloudTrail logs** from S3 (using `aws_connector`)
2. **Normalizes events** to Parquet format
3. **Builds feature matrix** for ML models
4. **Runs ML models** (Isolation Forest, LOF, Autoencoder, Ensemble)
5. **Generates alerts** automatically
6. **Updates RAG** vector store (optional)

**No need to manually run** `run_pipeline.py` or `run_models.py` anymore!

## Configuration

Edit `backend/config.yaml` to configure:

```yaml
aws:
  s3_bucket: "your-cloudtrail-logs-bucket"
  s3_prefix: "AWSLogs/123456789/CloudTrail/ap-south-1/"
  profile: "default"
  region: "ap-south-1"

pipeline:
  # Auto-run on backend startup
  run_on_startup: false
  
  # Default stages
  default_stages:
    - "ingest"
    - "features"
    - "models"
  
  # Scheduled runs (requires APScheduler)
  schedule:
    enabled: false
    cron: "0 */6 * * *"  # Every 6 hours
```

## Running the Backend

From the project root directory:

```bash
# Development mode with auto-reload
uvicorn backend.main:app --reload --port 8000

# Production mode
uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

The API will be available at:
- **API**: http://localhost:8000
- **Interactive docs**: http://localhost:8000/docs
- **Alternative docs**: http://localhost:8000/redoc

## API Endpoints

### Alerts

- `GET /api/alerts` - Get paginated alerts with filtering
- `GET /api/alerts/{user}/{window}` - Get enriched alert details
- `GET /api/alerts/filters` - Get available filter options
- `GET /api/alerts/summary/timeline` - Get alert timeline
- `GET /api/alerts/summary/top-users` - Get users with most alerts

### Pipeline Orchestration

- `POST /api/pipeline/run` - Trigger pipeline run (ingest CloudTrail, run models)
- `GET /api/pipeline/status` - Get current pipeline status
- `GET /api/pipeline/history` - Get pipeline execution history
- `GET /api/pipeline/config` - Get pipeline configuration
- `GET /api/pipeline/data-status` - Check data file status

### Statistics

- `GET /api/stats/overview` - Dashboard overview statistics
- `GET /api/stats/models` - ML model performance metrics
- `GET /api/stats/severity-distribution` - Alert severity breakdown
- `GET /api/stats/attack-distribution` - Attack type distribution

### RAG & Knowledge Base

- `POST /api/rag/query` - Query knowledge base (behavioral incidents, threat intel)
- `GET /api/rag/query` - Query via GET request
- `GTrigger pipeline to ingest CloudTrail logs

```bash
curl -X POST "http://localhost:8000/api/pipeline/run" \
  -H "Content-Type: application/json" \
  -d '{
    "s3_bucket": "my-cloudtrail-logs",
    "start_date": "2026-03-01",
    "end_date": "2026-03-04",
    "stages": ["ingest", "features", "models"],
    "run_async": true
  }'
```

### Check pipeline status

```bash
curl "http://localhost:8000/api/pipeline/status"
```

### ET /api/rag/playbooks` - List all incident response playbooks
- `GET /api/rag/playbooks/{id}` - Get specific playbook
- `GET /api/rag/techniques` - List MITRE ATT&CK techniques
- `GET /api/rag/techniques/{id}` - Get specific technique
- `GET /api/rag/collections` - Get available ChromaDB collections

## Data Requirements

The backend requires these data files to be present:

- `data/results/ensemble_alerts.csv` - Alert data (run `python run_models.py`)
- `data/results/*_results.json` - Model performance metrics
- `data/features/feature_matrix.parquet` - Feature matrix for enrichment
- `data/normalized/events_labeled.parquet` - Raw events for context
- `knowledge_base/playbooks.json` - Incident response playbooks
- `knowledge_base/mitre_techniques.json` - MITRE ATT&CK techniques
- `chroma_db/` - ChromaDB vector store (run RAG ingestion scripts)

## CORS

The API allows cross-origin requests from any origin (for development). 
In production, update the CORS settings in [main.py](main.py) to restrict origins:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-frontend-domain.com"],
    ...
)
```
